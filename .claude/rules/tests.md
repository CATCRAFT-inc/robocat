# Домен: tests/ — правила

## 🛑 Циклический импорт — главная ловушка

`bot.utils ↔ bot.bot ↔ bot.storage`. Первым ВСЕГДА импортируется `bot.bot` — `tests/conftest.py` делает это за всех (`import bot.bot` в конце, после фейкового env). Полное объяснение — в докстринге conftest. Что из этого следует:

- Не импортируй `bot.utils`/`bot.storage` на уровне модуля раньше, чем отработал conftest (в тестах — не проблема, conftest всегда первый; в скриптах/скиллах — начинай с `import bot.bot`).
- Не «чини» этот порядок в conftest без чтения его докстринга.

## Окружение

- `tests/conftest.py` подставляет фейковые env (`DISCORD_TOKEN`, ключи LLM, RCON) — тесты не требуют `.env` и НЕ ходят в сеть. Новая обязательная переменная окружения в коде → добавь фейк в `_FAKE_ENV`, иначе упадёт импорт.
- `pytest.ini`: `asyncio_mode = auto` — async-тесты пишутся без декораторов.

## Дисциплина

- Мокай все внешние сервисы: Discord API, LLM, RCON. Реальный вызов из теста = баг теста.
- Правка shared-функции (`utils`, `llm`, `flags`) → полный `pytest -q`, не точечный файл: точечный прогон пропускает регрессии в соседних тестах, дёргающих ту же функцию.
- Naming: `test_<действие>_<условие>_<ожидание>`.
- Перед любым push в master тесты обязаны быть зелёными (master = автодеплой на прод).
