# Домен: bot/ai/ — LLM-слой

## Инварианты

- **`llm.py` — единственная точка выхода в LLM API.** `engine.py`, тикеты и `/digest` ходят в модели только через него. Новый потребитель ИИ — тоже через `llm`, не через свой `AsyncClient`.
- **Ротация вендоров**: список в `data/ai_settings.yaml` (`vendors:`), у каждого `model` + `base_url` + `env` (имя переменной с ключом: `GEMINI`, `GROQ`, `OR`, `GHM`, `DS`) + `has_vision`. На ошибке вендора — кулдаун и переход к следующему. `extra_body` пробрасывается в запрос как есть.
- **`utility_model`** — отдельная дешёвая модель для структурного вывода (`llm.parse`) и фоновых задач (`llm.ask(use_utility=True)`).
- **Клиенты ленивые** — импорт `llm.py`/`embeddings.py` без сети и ключей ничего не запрашивает. Не ломай это: тесты и оффлайн-запуск на этом держатся.
- **`strip_thoughts()` обязателен для всех выходов ИИ** — выжимки, digest, саммари. Прецедент 04.07.2026 (0432c75): незакрытый `<thought>` утекал пользователям; вырезается и незакрытый тег.
- **Эмбеддинги**: `gemini-embedding-001`, 768-мерные. Используются для дедупа баг-репортов и поиска по вики (`wiki_search.py` → `data/wiki_index.json`, 9.5 MB, коммитится).
- **`data/ai_settings.yaml` коммитится** — промпты и конфиг вендоров да, ключи НИКОГДА (ключи только в `.env`).

## Тесты

- Реальные вызовы LLM/эмбеддингов в тестах запрещены — мокай `llm`. `tests/conftest.py` подставляет фейковые ключи, чтобы импорт не падал.
- Логика ротации и чанкинга уже покрыта (`test_llm.py`, `test_chunking.py`) — правишь ротацию → гоняй их полностью.
