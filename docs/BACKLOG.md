# BACKLOG — robocat

Отложенные находки код-ревью (issue #5, 18.07.2026). Баги и опасная мелочь
починены в ветке `claude/review-fixes`; здесь — то, что требует решения владельца
(деплой-пайплайн, дизайн-изменения) либо низкоприоритетно. Формат: `severity | файл | суть → рекомендация`.

## Деплой и CI (трогать деплой = риск для прода — на решение владельца)

- **MAJOR** | `.github/workflows/deploy.yml` | Деплой зелёный, даже если бот умер на старте: `systemctl restart` возвращает 0 сразу после запуска процесса, не дожидаясь готовности. → Добавить после рестарта `sleep 5 && systemctl is-active --quiet robocat` (или проверку по journalctl), чтобы упавший старт красил деплой.
- **MAJOR** | `deploy.yml` | Неатомарный деплой: `git pull` до `pip install`. Упавший pip оставляет новый код без зависимостей → внеплановый рестарт/ребут = crash-loop на «новый код + старые deps». → Ставить зависимости из fetch'нутого коммита ДО переключения дерева, либо `git pull` только после успешного `pip install`.
- **MAJOR** | `requirements.txt` | Только нижние границы версий, лок-файла нет: CI тестирует свежие версии, прод живёт на замороженных от даты создания venv, пересборка venv может подтянуть мажорный breaking-релиз disnake/openai. → `pip-compile` lock или верхние границы у `disnake`/`openai`.
- **MINOR** | `deploy.yml` | Нет `concurrency`-группы и `timeout-minutes` у deploy-джобы: два быстрых пуша деплоятся параллельно (рестарт посреди чужого pip install), зависший SSH держит ран до 6ч. → `concurrency: {group: deploy, cancel-in-progress: false}` + таймаут.
- **MINOR** | `deploy.yml:33` | `appleboy/ssh-action@v1.0.3` — мутабельный тег с доступом к прод-SSH-ключу. → Пин по полному commit SHA.
- **MINOR** | `deploy.yml` | `pytest`/`pytest-asyncio` без пинов в CI: breaking-релиз инструмента блокирует все деплои, в т.ч. срочный хотфикс. → Пины `pytest~=8` `pytest-asyncio~=1`.
- **MINOR** | `deploy.yml` | Деплой не запускает `data/db_init.py` — новая таблица + использующий код деплоятся зелёными, но в рантайме `no such table`. → `python data/db_init.py` перед рестартом (идемпотентен).

## Конфиг и тесты

- **MINOR** | `.env.example` | Разошёлся с кодом: нет LLM-ключей (`GEMINI`/`GROQ`/`OR`/`GHM`/`DS`), есть мёртвые `DATABASE_URL`/`WEB_TOKEN`/`MC_*`. При пересборке VDS по эталону AI-слой мёртв (`api_key=None`). → Синхронизировать с реально читаемыми переменными (сверить перед правкой — вдруг MC_* используется вне репо).
- **MINOR** | `tests/conftest.py` | `setdefault` оставляет реальные секреты из env разработчика в тестах: экспортированный настоящий `DISCORD_TOKEN` не заменится фейком. → Безусловная перезапись `os.environ[k]=v` (тесту настоящий токен не нужен).

## Ядро (низкоприоритетные баги)

- **MINOR** | `flag_commands.py` | `CheckFailure` глотается без ответа на интеракцию → не-админ видит «Приложение не отвечает». Тот же эффект у команд `admin.py` (нет error handler). → `inter.send("Нет прав", ephemeral=True)` в обработчике.
- **MINOR** | `flag_system.py:77` | Легитимное значение `+цифры` (телефон через `/flag_user`) трактуется как инкремент и искажается. Корень — общая семантика setFlag. → Отдельный API `incrementFlag` или явный маркер инкремента (в памяти уже экранировано локально).
- **MINOR** | `flag_commands.py` | `/get_user_flag`: значение >4000 символов (напр. `ai_summary`) роняет TextDisplay HTTP 400. → Обрезка значения перед отправкой.
- **MINOR** | `bot.py` on_member_join / `misc.py` | `get_channel` при cache-miss = None → лог «сейчас упадёт» и реальный AttributeError, welcome и снятие флага `left` теряются. → `return` после лога либо фолбэк `fetch_channel`.
- **MINOR** | `bot.py` on_ready | «Я тут!» шлётся на каждый reconnect-ready, не только при старте → индикатор рестарта бесполезен. → One-shot флаг.

## Хендлеры (низкоприоритетные баги)

- **MINOR** | `tickets/engine.py` | Нет guard'а от параллельного закрытия тикета двумя админами: второй постит мусорный архив + необработанный 404. → check-and-remove `created_by` до архивации.
- **MINOR** | `tickets/admin_ticket.py` | `raise` после залогированных закрытых ЛС — шум необработанных исключений (тикет уже создан). → Убрать `raise`.
- **MINOR** | `tickets/bugs.py` | `/clearbugs` без defer (протухает на десятках тредов) и молчит при отказе в доступе. → defer + ephemeral-ответ на отказ.
- **MINOR** | `tickets/bugs.py` | `rebuild_bug_index` сканирует вне лока: параллельно созданный баг затирается перезаписью индекса. → Держать лок на весь ребилд либо merge вместо перезаписи.
- **MINOR** | `rcon.py` | `command` + ответ RCON могут превысить 4000-символьный лимит V2 → 400 не пойман. → Обрезка ответа.
- **MINOR** | `digest.py` | `history(after=cutoff, limit=200, oldest_first=True)` берёт СТАРЕЙШИЕ 200 сообщений окна — свежие новости систематически теряются. → newest-first + reverse, либо больше лимит.
- **MINOR** | `catcraft_fm.py` | Блокирующий дисковый I/O (`os.listdir`, `TinyTag.get`) в event loop на каждый трек/`!очередь` — подвисания бота на медленном диске VDS. → `asyncio.to_thread`.
- **MINOR** | `punishments.py` | Ошибочные ветки `!мут` отвечают `ctx.author.send` без обработки `Forbidden` — при закрытых ЛС модератора команда молча исчезает. → try/except вокруг author.send + фолбэк в канал.
